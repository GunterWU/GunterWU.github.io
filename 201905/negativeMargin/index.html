margin负值简介

  1. margin属性值为负值是W3C标准认可的写法，不是hack
  2. DreamWare软件的设计视图不解析负margin值
  3. 不脱离文档流
  4. 兼容性良好（IE6在大多数情况下支持）
  5. 与设置正值不同，margin负值需要根据设置的方向以及元素是否浮动以及其定位方式来判断最终的行为

元素不浮动 && 设置定位

静态定位

  • position: static; 默认值
    margin-top/left:-20px;————该元素往top/left方向移动20px
    margin-bottom/right:-20px;————该元素本身不移动，其后面的元素往top/left方向移动20px并覆盖该元素
    width:auto时设置margin-left/right负值，该元素在对应方向上增加宽度，效果同于padding-left/right
    以上规则适用于兄弟元素，若为父子元素，则最后一个子元素设置负值margin-bottom/right时，父元素被该子元素覆盖，与兄弟元素覆盖顺序相反
    例如ul>li*5，最后一个li子元素会覆盖ul父元素，[去除ul最后一个li元素的border-bottom]是经典案例

相对定位

  • position: relative;
    margin-top/left:-20px;————该元素往top/left方向移动20px
    margin-bottom/right:-20px;————该元素本身不移动，其后面的元素往top/left方向移动20px并被该元素覆盖

绝对定位

  • position: absolute;
    margin-top/left:-20px;————该元素往top/left方向移动20px
    margin-bottom/right:-20px;————绝对定位元素脱离文档流，对后面的元素无影响

元素浮动 && 设置定位

静态定位

此时要依据元素的float和margin方向判断：

  • margin与float方向相同
    .elem {float:left/right;margin-left/right:-20px;}————该元素往浮动方向移动20px
  • margin与float方向相反
    .elem {float:left/right;margin-right/left:-20px;}————该元素本身不移动，其前面或后面的元素会往浮动方向移动20px并覆盖该元素，如.elem {float:right;margin-left:-20px;}位于该元素左边的右浮动元素则会向右移动20px，同时覆盖在该元素上

相对定位

元素设置浮动且设置position:relative;，元素行为和单独设置浮动是一样的

绝对定位

元素既设置浮动又绝对定位，这种情况是荒唐的
